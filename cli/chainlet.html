# chainlet

chainlet is the command used to launch chainlets onto a saga cluster. As we only support aws, we will deploy chainlets to the saga-eks cluster.

All chainlet namespaces are prefixed with saga-. For instance, chainlet test will have the kubernetes namespace saga-test.

$ saga chainlet
Chainlet-related commands

Usage:
  saga chainlet [command]

Available Commands:
  connect     Connects localhost to your chainlet
  delete      Deletes a chainlet
  faucet      Sends 1saga (1e18 asaga) to an address
  get         Get chainlet details
  get-pod     Get the pod name for a chainlet
  launch      Launches an EVM chainlet
  list        Lists the existing chainlets
  logs        Tails the pod logs for a chainlet

Flags:
  -h, --help   help for chainlet

Global Flags:
  -l, --loglevel string   Log level (default "info")

Use "saga chainlet [command] --help" for more information about a command.

# CRUD

You'll see a few CRUD-like actions, launch, get, and delete, all of which take one argument cluster-name, as well as list, which takes no arguments.

# faucet

faucet can be used to request 1 saga token to an address:

$ saga chainlet faucet --help
Sends 1saga (1e18 asaga) to an address

Usage:
  saga chainlet faucet [chainlet-name] [address] [flags]

# logs

logs can be used to tail logs for a chainlet.

$ saga chainlet logs --help  
Tails the pod logs for a chainlet

Usage:
  saga chainlet logs [chainlet-name] [flags]

Aliases:
  logs, log

# connect

connect can be used to connect a chainlet to your localhost.

$ saga chainlet connect --help
Connects localhost to your chainlet

Usage:
  saga chainlet connect [chainlet-name] [flags]
