# init

init initializes the saga cli and should always be run after installing or updating saga. This command is required before any of the other commands work.

When running saga init, you can pass the EC2 instance type as a flag to set the default aws instance type.

If you omit the instance-type parm when running saga init, the default sizing used is m5.2xlarge.

The minimum requirements to be met by the selected EC2 instance type are as follows:

  1. CPU - 1 vCPU
  2. Memory - 2 GB
  3. Storage - 8 GB

To control the size of the instance created, simply pass the --instance-type <EC2 instance type> like so:

saga init --instance-type t3.large

Note: The application validates the instance type passed to init based on the AWS region you have selected.

$ saga init --help
initializes the saga cli environment

Usage:
  saga init [flags]

Aliases:
  init, setup

Examples:
saga init

Flags:
  -h, --help                   help for init
      --instance-type string   saga init --instance-type=<EC2 instance type> (default "m5.2xlarge")
